# Definition of Done

A change is done when:

- [ ] The stated problem is solved.
- [ ] Scope matches the agreed plan.
- [ ] Tests cover new or changed behavior.
- [ ] Validation evidence is attached to the PR or handoff.
- [ ] Docs, examples, configs, and migrations are updated if needed.
- [ ] Risks and follow-ups are explicit.
- [ ] The change is ready to deploy or has a clear release plan.
